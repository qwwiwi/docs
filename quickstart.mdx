# Быстрый старт

Одна команда – и твой агент подключён к EdgeLab. Получи ключ в боте, добавь MCP-сервер и вызывай tools.

<Steps>

<Step title="Подключись к MCP-серверу">

Адрес один: `https://mcp.edgelab.su/mcp`

Два варианта подключения через Claude Code CLI:

```bash
# С API-ключом (получи в боте @InvestorEdgeLabbot):
claude mcp add --transport http -H "Authorization: Bearer el_live_твой_ключ" edgelab https://mcp.edgelab.su/mcp

# Без ключа (OAuth через Telegram -- нужен браузер):
claude mcp add --transport http edgelab https://mcp.edgelab.su/mcp
```

Для других клиентов -- см. Step 3.

</Step>

<Step title="Выбери способ авторизации">

Авторизация обязательна. Два способа:

| Способ | Что нужно | Когда использовать |
|---|---|---|
| API-ключ | Ключ от [@InvestorEdgeLabbot](https://t.me/InvestorEdgeLabbot) | Claude Desktop, Cursor, Windsurf, Claude Code с `-H` флагом |
| OAuth | Браузер + Telegram | Claude Code без ключа -- flow запускается автоматически |

После авторизации tier определяет набор tools: Free -- 18, Edge -- 39, Pro/VIP -- 63.

</Step>

<Step title="Добавь конфиг в свой клиент">

<CodeGroup>
```bash Claude Code с ключом
claude mcp add --transport http -H "Authorization: Bearer el_live_твой_ключ" edgelab https://mcp.edgelab.su/mcp
```

```bash Claude Code без ключа (OAuth)
claude mcp add --transport http edgelab https://mcp.edgelab.su/mcp
```

```json Claude Code (конфиг)
{
  "mcpServers": {
    "edgelab": {
      "type": "streamable-http",
      "url": "https://mcp.edgelab.su/mcp"
    }
  }
}
```

```json Claude Desktop
{
  "mcpServers": {
    "edgelab": {
      "url": "https://mcp.edgelab.su/mcp",
      "headers": {
        "Authorization": "Bearer el_live_ваш_ключ"
      }
    }
  }
}
```

```json Cursor / Windsurf
{
  "mcpServers": {
    "edgelab": {
      "url": "https://mcp.edgelab.su/mcp",
      "headers": {
        "x-api-key": "el_live_ваш_ключ"
      }
    }
  }
}
```
</CodeGroup>

<Tip>
**Claude Code** поддерживает оба способа. С ключом -- подключение мгновенное. Без ключа -- при первом запросе автоматически запустится OAuth через Telegram. Ключ получаешь в боте [@InvestorEdgeLabbot](https://t.me/InvestorEdgeLabbot).
</Tip>

</Step>

<Step title="Авторизуйся">

**Для Claude Code (OAuth):**
1. При первом вызове tool откроется браузер
2. На странице — кнопка **«Открыть Telegram»**
3. Перейди в бота, нажми START → готово

**Для остальных клиентов (API-ключ):**
1. Открой [@InvestorEdgeLabbot](https://t.me/InvestorEdgeLabbot)
2. Нажми START → бот выдаст ключ `el_live_...`
3. Добавь ключ в конфиг клиента

</Step>

<Step title="Сделай первый вызов">

После подключения MCP-сервера вызов tool происходит через клиента, а не через `curl`. В Claude Code просто попроси агента:

```text
Вызови tool list_knowledge с category=guide, per_page=5
```

Или задай вопрос на естественном языке:

```text
Покажи мне первые 5 гайдов с EdgeLab
```

Агент сам выберет нужный tool и передаст аргументы. Под капотом MCP-клиент отправит такой запрос:

```json
{
  "name": "list_knowledge",
  "arguments": {
    "category": "guide",
    "page": 1,
    "per_page": 5
  }
}
```

<Warning>
**Нельзя** вызывать MCP tools через `curl` или raw HTTP. MCP -- это не REST API, а streaming-протокол с авторизацией. Вызов всегда идёт через MCP-клиент (Claude Code, Claude Desktop, Cursor, Windsurf) после подключения.
</Warning>

Если всё подключено правильно, в ответе придёт список гайдов.

</Step>

</Steps>

## Tools на Free-тарифе (18 штук)

После авторизации пользователь с Free-тарифом получает:

```text
health, search_knowledge, get_knowledge, list_knowledge
list_categories, get_knowledge_children, list_recent_knowledge
get_sections, get_dashboard, get_faq, get_billing_plans
list_events, get_event, get_event_calendar
register, write_review, add_favorite, remove_favorite
```
