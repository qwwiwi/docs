# Knowledge

Knowledge -- это ядро EdgeLab: гайды, уроки, скиллы, воркшопы, usecases и live-записи.

## Доступные tools

| Tool | Доступ | Что делает |
|---|---|---|
| `search_knowledge` | Public | Поиск по базе |
| `get_knowledge` | Public | Получение конкретного материала |
| `list_knowledge` | Public | Список материалов с фильтрами |
| `list_categories` | Public | Список категорий |
| `get_knowledge_children` | Public | Дочерние элементы: шаги, prerequisites, tools, ссылки |
| `list_recent_knowledge` | Public | Свежие публикации |
| `get_sections` | Public | Секции и структурные блоки материала |

<Info>
Публичность tool не означает полный доступ ко всем материалам. Содержимое всё равно режется по тарифу.
</Info>

## Категории

```text
guide
lesson
skill
workshop
usecase
live
```

## `search_knowledge`

Подходит, когда агент знает тему, но не знает slug или точное название.

Типичные аргументы:

- `query`
- `category`
- `tier`
- `limit`

Пример:

```json
{
  "name": "search_knowledge",
  "arguments": {
    "query": "mcp server",
    "category": "guide",
    "limit": 5
  }
}
```

## `list_knowledge`

Нужен, когда нужен список по категории, tier или странице.

Типичные аргументы:

- `category`
- `tier`
- `page`
- `per_page`

## `get_knowledge`

Возвращает один материал по `slug_or_id`.

Если у пользователя нет нужного тарифа, tool обычно вернёт preview или ограниченную версию, а не полный закрытый контент.

## `get_knowledge_children`

Полезен, когда материал имеет внутреннюю структуру:

- шаги
- prerequisites
- список tools
- source URLs

Это особенно удобно для lesson/skill/workshop материалов.

## `list_recent_knowledge`

Лучший tool для фонового мониторинга новых публикаций.

Пример:

```json
{
  "name": "list_recent_knowledge",
  "arguments": {
    "limit": 10
  }
}
```

## `get_sections`

Подходит, когда агент хочет быстро разобрать структуру текста на крупные блоки до чтения полного материала.

## Практика

Хороший паттерн для агента:

```text
search_knowledge -> get_knowledge -> get_knowledge_children
```

или

```text
list_recent_knowledge -> get_knowledge -> get_sections
```
