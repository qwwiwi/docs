# Network endpoints

<Note>
Расширенные endpoints нетворкинга находятся в разработке. Базовый просмотр участников доступен через <a href="/api-reference/profile">GET /profile/members</a>. Ниже описана планируемая функциональность расширения.
</Note>

Endpoints для нетворкинга: создание запросов на сотрудничество, просмотр своих запросов и отклики на запросы других участников. Доступно на тарифах Pro и VIP.

---

## GET /profile/members (уже доступен)

Просмотр списка участников сообщества. Подробности -- в разделе [Profile endpoints](/api-reference/profile#get-profilemembers).

## POST /network/requests

<Info>
Coming soon -- этот endpoint появится в ближайшем обновлении API.
</Info>

Создать запрос на нетворкинг. Это способ найти участников для совместных проектов, обмена опытом или менторства.

**Scope:** `network:write` (Pro, VIP)

**Полный URL:** `POST https://platform.edgelab.su/api/v1/network/requests`

### Планируемые параметры запроса (body)

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `title` | string | Да | Заголовок запроса (макс. 100 символов) |
| `description` | string | Да | Описание: что ищешь, что предлагаешь (макс. 1000 символов) |
| `type` | string | Да | Тип: `collaboration`, `mentorship`, `feedback` |

### Планируемое поведение

- Создаёт публичный запрос, видимый другим участникам с тарифами Pro и VIP
- Другие участники могут откликнуться через endpoint interest
- Автор получает уведомление при каждом отклике

## GET /network/requests

<Info>
Coming soon -- этот endpoint появится в ближайшем обновлении API.
</Info>

Получить список своих запросов на нетворкинг.

**Scope:** `network:read` (Pro, VIP)

**Полный URL:** `GET https://platform.edgelab.su/api/v1/network/requests`

### Планируемые query-параметры

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `page` | integer | 1 | Номер страницы |
| `per_page` | integer | 20 | Количество на странице |
| `status` | string | -- | Фильтр по статусу: `active`, `closed` |

### Планируемая структура ответа

Каждый запрос будет содержать:

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Идентификатор запроса |
| `title` | string | Заголовок |
| `description` | string | Описание |
| `type` | string | Тип запроса |
| `status` | string | Статус: `active` или `closed` |
| `interests_count` | integer | Количество откликов |
| `created_at` | string | Дата создания (ISO 8601) |

## POST /network/requests/{id}/interest

<Info>
Coming soon -- этот endpoint появится в ближайшем обновлении API.
</Info>

Откликнуться на запрос нетворкинга другого участника.

**Scope:** `network:write` (Pro, VIP)

**Полный URL:** `POST https://platform.edgelab.su/api/v1/network/requests/{id}/interest`

### Планируемые параметры пути

| Параметр | Тип | Описание |
|---|---|---|
| `id` | string | Идентификатор запроса |

### Планируемые параметры запроса (body)

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `message` | string | Нет | Сопроводительное сообщение (макс. 500 символов) |

### Планируемое поведение

- Автор запроса получает уведомление об отклике
- Нельзя откликнуться на свой собственный запрос -- `422 Validation Error`
- Повторный отклик -- `409 Conflict`
