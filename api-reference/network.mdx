# Network

Нетворк в EdgeLab нужен, чтобы агент мог искать людей, находить совпадения и инициировать контакт.

## Доступные tools

| Tool | Доступ | Что делает |
|---|---|---|
| `list_members` | Pro/VIP | Возвращает список участников |
| `search_members` | Pro/VIP | Ищет участников по фильтрам и запросу |
| `get_matches` | Pro/VIP | Возвращает алгоритмические совпадения |
| `list_connect_requests` | Pro/VIP | Список входящих и исходящих connect-запросов |
| `send_connect_request` | Pro/VIP | Отправляет connect-запрос |
| `respond_connect_request` | Pro/VIP | Принимает или отклоняет запрос |

<Warning>
В общем тарифном описании network-функции присутствуют уже на Edge, но социальный MCP-контур с `list_members` и connect-tools в текущем наборе tools зафиксирован как `Pro/VIP`. Если продукт это изменит, править нужно одновременно и этот раздел, и таблицы тарифов.
</Warning>

## `list_members`

Подходит для обзора каталога участников и построения собственных shortlist'ов.

## `search_members`

Используй, когда агент ищет людей под конкретную задачу:

- по навыкам
- по интересам
- по роли
- по городу
- по текстовому запросу

## `get_matches`

Это самый удобный старт, если не хочется строить поиск вручную. Платформа уже отдает наиболее релевантные совпадения по профилю.

## `list_connect_requests`

Нужен для мониторинга состояния диалогов:

- кто отправил запрос
- кому уже отправили
- что ещё pending

## `send_connect_request`

Отправляет инициирующий запрос.

### Пример вызова

```json
{
  "name": "send_connect_request",
  "arguments": {
    "target_user_id": "user_123",
    "message": "Вижу совпадение по AI automation и crypto. Давай познакомимся."
  }
}
```

## `respond_connect_request`

Нужен для ответа на входящий запрос.

Типичный аргумент:

- id запроса
- решение: accept или reject

## Практика

Хороший flow:

```text
get_matches -> search_members -> send_connect_request -> list_connect_requests -> respond_connect_request
```
