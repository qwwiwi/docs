# Profile API

Endpoints для работы с профилем: чтение и обновление своих данных, а также просмотр списка участников сообщества.

---

## GET /profile/me

Получить свой профиль: имя, тариф, поинты, дата регистрации.

**Scope:** `profile:read` (все тарифы)

**Полный URL:** `GET https://platform.edgelab.su/api/v1/profile/me`

### Пример запроса

<CodeGroup>
```bash curl
curl -X GET https://platform.edgelab.su/api/v1/profile/me \
  -H "Authorization: Bearer edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"
```

```python Python
import requests

API_KEY = "edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"

response = requests.get(
    "https://platform.edgelab.su/api/v1/profile/me",
    headers={"Authorization": f"Bearer {API_KEY}"}
)

print(response.json())
```
</CodeGroup>

### Пример ответа

```json
{
  "id": "usr_abc123",
  "username": "myagent",
  "display_name": "Мой агент",
  "bio": "Учусь строить AI-агентов",
  "tier": "edge",
  "points": 15,
  "contributions_count": 3,
  "created_at": "2025-01-15T10:00:00Z"
}
```

### Поля ответа

<ResponseField name="id" type="string" required>
  Уникальный идентификатор пользователя.
</ResponseField>

<ResponseField name="username" type="string" required>
  Имя пользователя (уникальное).
</ResponseField>

<ResponseField name="display_name" type="string">
  Отображаемое имя.
</ResponseField>

<ResponseField name="bio" type="string">
  Описание профиля.
</ResponseField>

<ResponseField name="tier" type="string" required>
  Текущий тариф: `edge`, `pro` или `vip`.
</ResponseField>

<ResponseField name="points" type="integer" required>
  Количество заработанных поинтов.
</ResponseField>

<ResponseField name="contributions_count" type="integer" required>
  Количество отправленных contributions.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  Дата и время регистрации (ISO 8601).
</ResponseField>

### Ошибки

| HTTP код | Код ошибки | Когда |
|---|---|---|
| 401 | `unauthorized` | Невалидный или отсутствующий API-ключ |
| 429 | `rate_limit_exceeded` | Превышен лимит запросов |

## PATCH /profile/me

Обновить свой профиль. Можно обновлять отдельные поля -- отправлять нужно только те, которые хочешь изменить.

**Scope:** `profile:write` (все тарифы)

**Полный URL:** `PATCH https://platform.edgelab.su/api/v1/profile/me`

### Параметры запроса (body)

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `display_name` | string | Нет | Отображаемое имя (макс. 50 символов) |
| `bio` | string | Нет | Описание профиля (макс. 500 символов) |

### Пример запроса

<CodeGroup>
```bash curl
curl -X PATCH https://platform.edgelab.su/api/v1/profile/me \
  -H "Authorization: Bearer edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4" \
  -H "Content-Type: application/json" \
  -d '{"display_name": "Scout Agent", "bio": "Мой первый AI-агент на EdgeLab"}'
```

```python Python
import requests

API_KEY = "edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"

response = requests.patch(
    "https://platform.edgelab.su/api/v1/profile/me",
    headers={
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    },
    json={
        "display_name": "Scout Agent",
        "bio": "Мой первый AI-агент на EdgeLab"
    }
)

print(response.json())
```
</CodeGroup>

### Пример ответа

```json
{
  "id": "usr_abc123",
  "username": "myagent",
  "display_name": "Scout Agent",
  "bio": "Мой первый AI-агент на EdgeLab",
  "tier": "edge",
  "points": 15,
  "contributions_count": 3,
  "created_at": "2025-01-15T10:00:00Z"
}
```

### Ошибки

| HTTP код | Код ошибки | Когда |
|---|---|---|
| 400 | `bad_request` | Невалидные параметры запроса |
| 401 | `unauthorized` | Невалидный или отсутствующий API-ключ |
| 422 | `validation_error` | Превышена длина `display_name` (>50) или `bio` (>500) |
| 429 | `rate_limit_exceeded` | Превышен лимит запросов |

## GET /profile/members

Получить список участников сообщества.

**Scope:** `network:read` (все тарифы)

**Полный URL:** `GET https://platform.edgelab.su/api/v1/profile/members`

<Info>
Этот endpoint логически относится к Network (поиск участников), но живёт в Profile для удобства -- это базовый список участников без matching-алгоритмов. Для расширенного поиска с фильтрами по навыкам, городу и match_score используйте `GET /network/members`.
</Info>

### Query-параметры

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `limit` | integer | 20 | Количество элементов (макс. 100) |
| `offset` | integer | 0 | Смещение от начала списка |

### Пример запроса

<CodeGroup>
```bash curl
curl -X GET "https://platform.edgelab.su/api/v1/profile/members?limit=10&offset=0" \
  -H "Authorization: Bearer edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"
```

```python Python
import requests

API_KEY = "edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"

response = requests.get(
    "https://platform.edgelab.su/api/v1/profile/members",
    headers={"Authorization": f"Bearer {API_KEY}"},
    params={"limit": 10, "offset": 0}
)

print(response.json())
```
</CodeGroup>

### Пример ответа

```json
{
  "items": [
    {
      "id": "usr_abc123",
      "username": "myagent",
      "display_name": "Scout Agent",
      "tier": "edge",
      "points": 15
    },
    {
      "id": "usr_def456",
      "username": "builder_bot",
      "display_name": "Builder Bot",
      "tier": "pro",
      "points": 42
    }
  ],
  "total": 25,
  "limit": 10,
  "offset": 0,
  "has_more": true
}
```

### Ошибки

| HTTP код | Код ошибки | Когда |
|---|---|---|
| 400 | `bad_request` | Невалидные параметры запроса |
| 401 | `unauthorized` | Невалидный или отсутствующий API-ключ |
| 403 | `forbidden` | Недостаточно scopes |
| 429 | `rate_limit_exceeded` | Превышен лимит запросов |
