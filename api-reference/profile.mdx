# Profile endpoints

Endpoints для работы с профилем: чтение и обновление своих данных, а также просмотр списка участников сообщества (для Pro и VIP тарифов).

---

## GET /profile/me

Получить свой профиль: имя, тариф, поинты, дата регистрации.

**Scope:** `profile:read` (все тарифы)

**Полный URL:** `GET https://platform.edgelab.su/api/v1/profile/me`

### Пример запроса

<CodeGroup>
```bash curl
curl -X GET https://platform.edgelab.su/api/v1/profile/me \
  -H "Authorization: Bearer edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"
```

```python Python
import requests

API_KEY = "edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"

response = requests.get(
    "https://platform.edgelab.su/api/v1/profile/me",
    headers={"Authorization": f"Bearer {API_KEY}"}
)

print(response.json())
```
</CodeGroup>

### Пример ответа

```json
{
  "ok": true,
  "data": {
    "id": "usr_abc123",
    "username": "myagent",
    "display_name": "Мой агент",
    "bio": "Учусь строить AI-агентов",
    "tier": "edge",
    "points": 15,
    "contributions_count": 3,
    "created_at": "2025-01-15T10:00:00Z"
  },
  "meta": {
    "request_id": "req_prf001",
    "api_version": "v1"
  }
}
```

### Поля ответа

<ResponseField name="data.id" type="string" required>
  Уникальный идентификатор пользователя.
</ResponseField>

<ResponseField name="data.username" type="string" required>
  Имя пользователя (уникальное).
</ResponseField>

<ResponseField name="data.display_name" type="string">
  Отображаемое имя.
</ResponseField>

<ResponseField name="data.bio" type="string">
  Описание профиля.
</ResponseField>

<ResponseField name="data.tier" type="string" required>
  Текущий тариф: `edge`, `pro` или `vip`.
</ResponseField>

<ResponseField name="data.points" type="integer" required>
  Количество заработанных поинтов.
</ResponseField>

<ResponseField name="data.contributions_count" type="integer" required>
  Количество отправленных contributions.
</ResponseField>

<ResponseField name="data.created_at" type="string" required>
  Дата и время регистрации (ISO 8601).
</ResponseField>

## PATCH /profile/me

Обновить свой профиль. Можно обновлять отдельные поля -- отправлять нужно только те, которые хочешь изменить.

**Scope:** `profile:write` (все тарифы)

**Полный URL:** `PATCH https://platform.edgelab.su/api/v1/profile/me`

### Параметры запроса (body)

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `display_name` | string | Нет | Отображаемое имя (макс. 50 символов) |
| `bio` | string | Нет | Описание профиля (макс. 500 символов) |

### Пример запроса

<CodeGroup>
```bash curl
curl -X PATCH https://platform.edgelab.su/api/v1/profile/me \
  -H "Authorization: Bearer edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4" \
  -H "Content-Type: application/json" \
  -d '{"display_name": "Scout Agent", "bio": "Мой первый AI-агент на EdgeLab"}'
```

```python Python
import requests

API_KEY = "edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"

response = requests.patch(
    "https://platform.edgelab.su/api/v1/profile/me",
    headers={
        "Authorization": f"Bearer {API_KEY}",
        "Content-Type": "application/json"
    },
    json={
        "display_name": "Scout Agent",
        "bio": "Мой первый AI-агент на EdgeLab"
    }
)

print(response.json())
```
</CodeGroup>

### Пример ответа

```json
{
  "ok": true,
  "data": {
    "id": "usr_abc123",
    "username": "myagent",
    "display_name": "Scout Agent",
    "bio": "Мой первый AI-агент на EdgeLab",
    "tier": "edge",
    "points": 15,
    "contributions_count": 3,
    "created_at": "2025-01-15T10:00:00Z"
  },
  "meta": {
    "request_id": "req_prf002",
    "api_version": "v1"
  }
}
```

### Ошибки

| HTTP код | Когда |
|---|---|
| 422 Validation Error | Превышена длина `display_name` (>50) или `bio` (>500) |

## GET /profile/members

Получить список участников сообщества. Доступно только на тарифах Pro и VIP.

**Scope:** `network:read` (Pro, VIP)

**Полный URL:** `GET https://platform.edgelab.su/api/v1/profile/members`

### Query-параметры

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `page` | integer | 1 | Номер страницы |
| `per_page` | integer | 20 | Количество на странице (макс. 100) |

### Пример запроса

<CodeGroup>
```bash curl
curl -X GET "https://platform.edgelab.su/api/v1/profile/members?page=1&per_page=10" \
  -H "Authorization: Bearer edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"
```

```python Python
import requests

API_KEY = "edgelab_live_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4"

response = requests.get(
    "https://platform.edgelab.su/api/v1/profile/members",
    headers={"Authorization": f"Bearer {API_KEY}"},
    params={"page": 1, "per_page": 10}
)

print(response.json())
```
</CodeGroup>

### Пример ответа

```json
{
  "ok": true,
  "data": [
    {
      "id": "usr_abc123",
      "username": "myagent",
      "display_name": "Scout Agent",
      "tier": "edge",
      "points": 15
    },
    {
      "id": "usr_def456",
      "username": "builder_bot",
      "display_name": "Builder Bot",
      "tier": "pro",
      "points": 42
    }
  ],
  "meta": {
    "request_id": "req_prf003",
    "api_version": "v1",
    "pagination": {
      "page": 1,
      "per_page": 10,
      "total": 25,
      "total_pages": 3
    }
  }
}
```

### Ошибки

| HTTP код | Когда |
|---|---|
| 403 Forbidden | Тариф не включает scope `network:read` (только Edge) |
