# API Keys

Раздел про auth tools. Здесь больше нет REST endpoint'ов `/auth/...`. Всё делается через MCP tools.

## Доступные tools

| Tool | Доступ | Что делает |
|---|---|---|
| `get_auth_me` | Edge+ | Показывает, какой ключ сейчас используется и какие у него scopes |
| `create_api_key` | Edge+ | Создаёт новый ключ |
| `list_api_keys` | Edge+ | Возвращает список активных ключей |
| `rotate_api_key` | Edge+ | Меняет старый ключ на новый |
| `delete_api_key` | Edge+ | Удаляет ключ |

<Info>
Формат ключа всегда такой: `el_live_1a8c25f8b7165604`
</Info>

## `create_api_key`

Создаёт новый ключ для агента или отдельного runtime.

### Аргументы

| Поле | Тип | Обязательно | Описание |
|---|---|---|---|
| `name` | string | нет | Человеческое имя ключа |

### Пример вызова

```json
{
  "name": "create_api_key",
  "arguments": {
    "name": "Scout agent"
  }
}
```

### Что вернётся

- полный новый ключ
- его prefix
- дата создания
- scopes, унаследованные от текущего тарифа

<Warning>
Полный ключ показывается только в момент создания. Потом ты увидишь только prefix и метаданные.
</Warning>

## `list_api_keys`

Возвращает все активные ключи текущего пользователя.

### Что удобно смотреть

- `prefix`
- `name`
- `created_at`
- `last_used_at`
- `scopes`

Используй этот tool для регулярного аудита: если видишь неожиданный `last_used_at`, это повод проверить, не утёк ли ключ.

## `rotate_api_key`

Ротация заменяет старый ключ на новый за один вызов.

### Аргументы

| Поле | Тип | Обязательно | Описание |
|---|---|---|---|
| `prefix` | string | да | Prefix ключа, который нужно заменить |

### Пример вызова

```json
{
  "name": "rotate_api_key",
  "arguments": {
    "prefix": "el_live_1a8c25f8"
  }
}
```

### Что важно помнить

- старый ключ перестаёт работать сразу
- новый ключ нужно сразу разнести по агентам и конфигам

## `delete_api_key`

Удаляет ключ без ротации.

### Аргументы

| Поле | Тип | Обязательно | Описание |
|---|---|---|---|
| `prefix` | string | да | Prefix ключа, который нужно удалить |

Используй `delete_api_key`, если ключ больше не нужен или уже скомпрометирован.

## `get_auth_me`

Показывает контекст текущей авторизации:

- кто владелец ключа
- какой тариф активен
- какие scopes реально доступны
- какой prefix используется

Это удобный диагностический вызов после подключения нового MCP-клиента.

## Практика

Рекомендованный цикл для нормальной эксплуатации:

1. `create_api_key` для каждого отдельного агента
2. `list_api_keys` для периодического аудита
3. `rotate_api_key`, если ключ засветился
4. `delete_api_key`, если ключ больше не нужен
5. `get_auth_me`, если нужно быстро проверить текущий контекст
