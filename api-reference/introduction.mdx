# Обзор MCP

EdgeLab использует MCP, а не REST API. Поэтому здесь нет «базового URL + endpoint'ы + методы». Есть один MCP-сервер и набор tools.

## Точка подключения

Адрес MCP-сервера: `https://mcp.edgelab.su/mcp`

Для подключения через Claude Code CLI:

```bash
claude mcp add edgelab-mcp --transport http https://mcp.edgelab.su/mcp
```

Для других клиентов (Claude Desktop, Cursor, Windsurf) -- смотри [гайд по подключению](/guides/connect-agent).

## Как выглядит вызов

MCP-клиент сам показывает tools в интерфейсе. Ты просто формулируешь задачу естественным языком, а клиент сам подбирает нужный tool и параметры:

```text
Найди на EdgeLab материалы про MCP сервер -- покажи 5 результатов
```

Под капотом клиент отправит MCP-серверу такой tool call:

```json
{
  "name": "search_knowledge",
  "arguments": {
    "query": "mcp server",
    "limit": 5
  }
}
```

<Warning>
**Не пытайся** вызывать tools через `curl` к `https://mcp.edgelab.su/mcp`. MCP -- это streaming-протокол с OAuth/API-key аутентификацией, не REST. Raw HTTP-запросы вернут `401 invalid_token`. Всегда используй MCP-клиент (Claude Code, Claude Desktop, Cursor, Windsurf).
</Warning>

## Аутентификация

- Без ключа доступны public tools
- С ключом доступны personal, social и admin tools
- Формат ключа: `el_live_` + `16` hex-символов
- Передача: `Authorization: Bearer el_live_...` или `x-api-key: el_live_...`

Подробнее смотри в [Аутентификации](/authentication).

## Как читать разделы ниже

Каждый файл в `api-reference/` теперь описывает не REST endpoint'ы, а MCP tools:

- что делает tool
- кому он доступен
- какие аргументы принимает
- какой тип ответа ожидать

Если ты ищешь старые `/api/v1/...` пути, их больше нет в актуальной модели доступа.
