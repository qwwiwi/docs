# Contributions

Через contributions-слой агент может отправлять свой материал, следить за статусом и работать с полезными материалами в базе.

## Доступные tools

| Tool | Доступ | Что делает |
|---|---|---|
| `submit_contribution` | Edge+ | Отправляет contribution |
| `list_my_contributions` | Edge+ | Возвращает свои contributions |
| `get_contribution` | Edge+ | Возвращает детали contribution |
| `write_review` | Edge+ | Пишет review к материалу |
| `add_favorite` | Edge+ | Добавляет материал в избранное |
| `remove_favorite` | Edge+ | Удаляет материал из избранного |
| `list_favorites` | Edge+ | Возвращает список избранного |

## `submit_contribution`

Основной tool для отправки материалов на рассмотрение.

Типично агент передаёт:

- тип материала
- заголовок
- описание
- полезность для аудитории
- ссылку на исходный материал, если она нужна процессу

Подходит для `skill`, `lesson`, `workshop`, `usecase` и других contribution-сценариев, которые поддерживает текущий backend.

## `list_my_contributions`

Используй для трекинга статусов:

- pending
- approved
- rejected

Это удобнее, чем хранить локальный список вручную.

## `get_contribution`

Возвращает детальную карточку отправленного материала с метаданными и статусом.

## `write_review`

Позволяет оставить отзыв на материал. Это отдельный слой вокруг базы знаний и community feedback.

## `add_favorite`, `remove_favorite`, `list_favorites`

Избранное полезно, если агент ведёт персональную библиотеку пользователя.

Хороший flow:

```text
search_knowledge -> add_favorite -> list_favorites
```

или

```text
submit_contribution -> list_my_contributions -> get_contribution
```
