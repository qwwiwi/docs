# Support API

Поддержка пользователей EdgeLab работает через два канала:

- **Telegram-бот:** [@EdgeLabSupportBot](https://t.me/EdgeLabSupportBot) -- быстрый доступ через привычный интерфейс
- **API:** Программный доступ для интеграций и автоматизации

<Info>
**FAQ Flow:** Перед созданием тикета система проверяет FAQ. Если ответ найден -- отдаёт сразу. Если нет -- предлагает создать тикет.
</Info>

Все тикеты обрабатываются без приоритетов -- каждый пользователь равен. Уведомления приходят через бот и колокольчик на платформе.

---

## GET /support/faq

Получить FAQ. Публичный endpoint -- авторизация не требуется.

**Scope:** Публичный (без авторизации)  
**URL:** `GET https://platform.edgelab.su/api/v1/support/faq`

### Query-параметры

<ParamField query="category" type="string">
  Фильтр: `account`, `billing`, `api`, `platform`, `agents`
</ParamField>

<ParamField query="q" type="string">
  Поиск по тексту вопросов и ответов
</ParamField>

### Пример ответа

```json
{
  "ok": true,
  "data": [
    {
      "id": "faq_001",
      "category": "api",
      "question": "Как получить API-ключ?",
      "answer": "Перейдите в Settings → API Keys и нажмите «Создать ключ». Ключ отображается один раз -- сохраните его.",
      "helpful_count": 42
    }
  ],
  "meta": { "request_id": "req_s001", "api_version": "v1", "total": 1 }
}
```

## POST /support/tickets

Создать тикет в службу поддержки.

**Scope:** `support:write` (все тарифы)  
**URL:** `POST https://platform.edgelab.su/api/v1/support/tickets`

| Заголовок | Значение |
|-----------|----------|
| `Authorization` | `Bearer edgelab_live_...` |
| `Content-Type` | `application/json` |

<ParamField body="subject" type="string" required>
  Тема тикета (макс. 150 символов)
</ParamField>

<ParamField body="message" type="string" required>
  Описание проблемы (макс. 2000 символов)
</ParamField>

<ParamField body="category" type="string" required>
  Категория: `account`, `billing`, `api`, `platform`, `agents`
</ParamField>

### Пример запроса

```bash
curl -X POST https://platform.edgelab.su/api/v1/support/tickets \
  -H "Authorization: Bearer edgelab_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "subject": "API возвращает 429 при низкой нагрузке",
    "message": "Получаю 429 при 50 req/min на Pro. Лимит должен быть 500.",
    "category": "api"
  }'
```

### Пример ответа

```json
{
  "ok": true,
  "data": {
    "id": "tkt_7d2f1a",
    "subject": "API возвращает 429 при низкой нагрузке",
    "category": "api",
    "status": "open",
    "author": { "user_id": "usr_abc123", "username": "developer42" },
    "created_at": "2026-03-21T14:15:00Z"
  },
  "meta": { "request_id": "req_s002", "api_version": "v1" }
}
```

| HTTP код | Когда |
|----------|-------|
| `401` | Невалидный или отсутствующий API-ключ |
| `422` | Невалидные данные: пустой subject, неизвестная category |

**URL:** `GET https://platform.edgelab.su/api/v1/support/tickets`

    "created_at": "2026-03-21T14:15:00Z",

  },

## POST /support/tickets/{id}/reply

Ответить в существующем тикете.

**Scope:** `support:write` (все тарифы)  
**URL:** `POST https://platform.edgelab.su/api/v1/support/tickets/{id}/reply`

<ParamField path="id" type="string" required>
  Идентификатор тикета
</ParamField>

<ParamField body="message" type="string" required>
  Текст ответа (макс. 2000 символов)
</ParamField>

### Пример ответа

```json
{
  "ok": true,
  "data": {
    "id": "reply_02",
    "ticket_id": "tkt_7d2f1a",
    "from": "user",
    "message": "Проблема решена, лимиты работают корректно. Спасибо!",
    "created_at": "2026-03-21T16:00:00Z"
  },
  "meta": { "request_id": "req_s005", "api_version": "v1" }
}
```

| HTTP код | Когда |
|----------|-------|
| `401` | Невалидный API-ключ |
| `404` | Тикет не найден |
| `422` | Пустое сообщение или тикет закрыт |

## POST /support/call-request

Запросить персональный созвон с Даши. Только тариф VIP.

**Scope:** `support:write` + тариф VIP  
**URL:** `POST https://platform.edgelab.su/api/v1/support/call-request`

<Warning>
Доступно только пользователям на тарифе **VIP**. Другие тарифы получат `403 Forbidden`.
</Warning>

<ParamField body="topic" type="string" required>
  Тема созвона (макс. 200 символов)
</ParamField>

<ParamField body="preferred_time" type="string">
  Предпочтительное время (ISO 8601)
</ParamField>

<ParamField body="notes" type="string">
  Дополнительные заметки (макс. 500 символов)
</ParamField>

### Пример запроса

```bash
curl -X POST https://platform.edgelab.su/api/v1/support/call-request \
  -H "Authorization: Bearer edgelab_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "Интеграция AI-агентов в production workflow",
    "preferred_time": "2026-03-25T15:00:00Z",
    "notes": "Обсудить архитектуру multi-agent системы"
  }'
```

### Пример ответа

```json
{
  "ok": true,
  "data": {
    "id": "call_3e9b1c",
    "topic": "Интеграция AI-агентов в production workflow",
    "status": "requested",
    "preferred_time": "2026-03-25T15:00:00Z",
    "created_at": "2026-03-21T16:30:00Z"
  },
  "meta": { "request_id": "req_s006", "api_version": "v1" }
}
```

| HTTP код | Когда |
|----------|-------|
| `401` | Невалидный API-ключ |
| `403` | Тариф пользователя не VIP |
| `422` | Пустая тема |

<Tip>
При ответе на тикет или изменении статуса созвона вы получите уведомление через @EdgeLabSupportBot и колокольчик на платформе.
</Tip>
