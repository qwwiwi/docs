# Подключение агента к EdgeLab

Если коротко: твой агент не ходит в REST endpoint'ы. Он подключается к MCP-серверу `https://mcp.edgelab.su/mcp` и вызывает tools.

## Что понадобится

- MCP-клиент: Claude Code, Claude Desktop, Cursor, Windsurf или свой runtime
- Telegram-бот [@InvestorEdgeLabbot](https://t.me/InvestorEdgeLabbot) — для регистрации и получения ключа
- Ключ `el_live_...` или OAuth через Telegram — для личных tools

## Два пути авторизации

| Путь | Для кого | Как работает |
|---|---|---|
| **OAuth через Telegram** | Claude Code и клиенты без заголовков | Автоматически при первом запросе. Подтверждение в Telegram-боте |
| **API-ключ** | Claude Desktop, Cursor, Windsurf | Ключ от бота → в заголовок конфига |

<Tip>
Claude Code поддерживает OAuth — ключ не нужен. Просто подключи MCP и подтверди через Telegram при первом запросе.
</Tip>

## Схема доступа

| Уровень | Нужен ключ/OAuth | Что получает агент |
|---|:---:|---|
| Free | -- | Публичные guides и FAQ |
| Edge | + | Профиль, прогресс, achievements, points, billing, support, notifications, events, network, contributions, referrals |
| Pro | + | Всё из Edge + `events:write`, usecases, priority support |
| VIP | + | Всё из Pro + rate limit 300/min + early access |

## Шаг за шагом

<Steps>

<Step title="Добавь MCP-конфиг">

<CodeGroup>
```bash Claude Code (CLI)
claude mcp add edgelab-mcp --transport http https://mcp.edgelab.su/mcp
```

```json Claude Code (конфиг)
{
  "mcpServers": {
    "edgelab": {
      "type": "streamable-http",
      "url": "https://mcp.edgelab.su/mcp"
    }
  }
}
```

```json Claude Desktop (с ключом)
{
  "mcpServers": {
    "edgelab": {
      "url": "https://mcp.edgelab.su/mcp",
      "headers": {
        "Authorization": "Bearer el_live_ваш_ключ"
      }
    }
  }
}
```

```json Cursor / Windsurf (с ключом)
{
  "mcpServers": {
    "edgelab": {
      "url": "https://mcp.edgelab.su/mcp",
      "headers": {
        "x-api-key": "el_live_ваш_ключ"
      }
    }
  }
}
```
</CodeGroup>

</Step>

<Step title="Авторизуйся">

**Claude Code (OAuth через Telegram):**
При первом вызове tool откроется браузер → страница с кнопкой «Открыть Telegram» → подтверди в боте → готово. [Подробнее](/guides/oauth-connect)

**Остальные клиенты (API-ключ):**
Открой [@InvestorEdgeLabbot](https://t.me/InvestorEdgeLabbot) → START → бот выдаст ключ → добавь в конфиг.

</Step>

<Step title="Проверь public tools">

Задачу агенту ставь естественным языком -- MCP-клиент сам выберет tool. Например:

```text
Вызови tool health чтобы проверить EdgeLab MCP
```

или:

```text
Покажи первые 3 гайда из EdgeLab
```

Под капотом клиент сделает tool call:

```json
{ "name": "health", "arguments": {} }
```

```json
{
  "name": "list_knowledge",
  "arguments": { "category": "guide", "page": 1, "per_page": 3 }
}
```

<Warning>
Вызывать MCP tools через `curl` или raw HTTP нельзя -- получишь `401 invalid_token`. MCP работает только через клиент (Claude Code, Claude Desktop, Cursor, Windsurf).
</Warning>

</Step>

<Step title="Проверь personal tools">

После авторизации вызови что-то из личного контура:

```json
{
  "name": "get_user_profile",
  "arguments": {}
}
```

Или:

```json
{
  "name": "get_dashboard",
  "arguments": {}
}
```

</Step>

</Steps>

## Полезные группы tools

### Knowledge

```text
search_knowledge, get_knowledge, list_knowledge
list_categories, get_knowledge_children
list_recent_knowledge, get_sections
```

### Profile и progress

```text
get_user_profile, get_full_profile, update_profile
link_telegram, track_progress, get_progress, get_progress_history
```

### Events

```text
list_events, get_event, get_event_calendar
rsvp_event, cancel_rsvp, propose_event, list_proposals
```

### Network

```text
list_members, search_members, get_matches
list_connect_requests, send_connect_request, respond_connect_request
```

### Support и notifications

```text
create_ticket, list_tickets, get_ticket, send_ticket_message
list_notifications, mark_notification_read, get_unread_count
```

## Рабочий паттерн агента

Обычно flow такой:

1. Агент ищет данные через `list_*` или `search_*`
2. Потом добирает детали через `get_*`
3. После этого делает действие: `rsvp_event`, `update_profile`, `create_ticket`, `submit_contribution`

Это надёжнее, чем пытаться «угадывать» id или контекст.

## Cron и фоновая автоматизация

Если агент работает по расписанию, ориентируйся на MCP tools:

| Частота | Tool | Зачем |
|---|---|---|
| Каждые 10 минут | `get_unread_count` | Проверить новые уведомления |
| Каждое утро | `list_recent_knowledge` | Забрать свежие материалы |
| Каждое утро | `list_events` | Проверить ближайшие события |
| После урока | `track_progress` | Обновить прогресс |
| 1–2 раза в день | `get_matches` | Проверить новые совпадения в нетворке |

## Rate limit и ошибки

| Тариф | Лимит |
|---|---:|
| Free | 30/min |
| Edge | 60/min |
| Pro | 120/min |
| VIP | 300/min |

Если агент получает `rate_limit_exceeded`, добавь backoff и не делай агрессивный polling.
