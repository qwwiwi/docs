# Подключение агента к EdgeLab

Если коротко: твой агент не ходит в REST endpoint'ы. Он подключается к MCP-серверу `https://mcp.edgelab.su/mcp` и вызывает tools.

## Что понадобится

- MCP-клиент: Claude Desktop, Cursor, Windsurf или свой runtime
- Ключ `el_live_...`, если нужны личные tools
- Telegram-бот [@InvestorEdgeLabbot](https://t.me/InvestorEdgeLabbot), если нужен personal-доступ

## Схема доступа

| Уровень | Нужен ключ | Что получает агент |
|---|:---:|---|
| Free | -- | Публичные guides и FAQ |
| Edge | + | Профиль, прогресс, achievements, points, billing, support, notifications, events, network, contributions, referrals |
| Pro | + | Всё из Edge + `events:write`, usecases, priority support |
| VIP | + | Всё из Pro без referrals, но с большими лимитами и early access |

<Warning>
VIP сейчас описан как модель доступа, но сами планы выключены. Не обещай пользователям, что его можно оформить прямо сейчас.
</Warning>

## Рабочий паттерн агента

Обычно flow такой:

1. Агент ищет данные через `list_*` или `search_*`
2. Потом добирает детали через `get_*`
3. После этого делает действие: `rsvp_event`, `update_profile`, `create_ticket`, `submit_contribution`

Это надёжнее, чем пытаться «угадывать» id или контекст.

## Cron и фоновая автоматизация

Если у тебя агент работает по расписанию, ориентируйся на MCP tools, а не на HTTP endpoint'ы.

| Частота | Tool | Зачем |
|---|---|---|
| Каждые 10 минут | `get_unread_count` | Быстро понять, появились ли новые уведомления |
| Каждое утро | `list_recent_knowledge` | Забрать свежие материалы |
| Каждое утро | `list_events` | Проверить ближайшие события |
| После завершения урока | `track_progress` | Обновить прогресс |
| 1-2 раза в день | `get_matches` | Проверить новые совпадения в нетворке |

Пример логики cron:

```text
1. get_unread_count
2. если unread > 0 -> list_notifications
3. list_recent_knowledge
4. list_events
5. если есть важное -> уведомить пользователя
```

## Rate limit и ошибки

Лимиты зависят от тарифа:

- Free: `30/min`
- Edge: `60/min`
- Pro: `120/min`
- VIP: `300/min`

Если агент получает `rate_limit_exceeded`, добавь backoff и не делай агрессивный polling.
