# Как стать контрибьютором

Контрибьюторы усиливают EdgeLab реальными рабочими материалами: skills, lessons, workshops, usecases, гайды и другие практические сценарии.

<Info>
Contribution-поток в MCP доступен с тарифа `Edge`.
</Info>

## Что считается хорошим contribution

Хороший contribution:

- решает конкретную задачу
- объясняет, кому и зачем он нужен
- показывает рабочий процесс, а не абстрактную идею
- его можно повторить

## Что можно отправлять

| Тип | Что это |
|---|---|
| `skill` | Навык или интеграция для агента |
| `lesson` | Обучающий материал |
| `workshop` | Практический разбор |
| `usecase` | Сценарий для реальной задачи |
| `guide` | Пошаговый гайд |

## Как отправить

Вместо старых REST-запросов используй MCP tool `submit_contribution`.

### Пример

```json
{
  "name": "submit_contribution",
  "arguments": {
    "type": "skill",
    "title": "Мониторинг CoinGecko для агента",
    "description": "Скилл для получения цен, базовой аналитики и алертов",
    "problem_solved": "Нет простого способа быстро получать крипто-данные в агентском workflow",
    "target_audience": "builders, researchers, traders",
    "github_url": "https://github.com/username/coingecko-skill"
  }
}
```

После отправки материал получает статус `pending`.

## Как следить за статусом

Используй:

- `list_my_contributions`
- `get_contribution`

Пример рабочего цикла:

```text
submit_contribution -> list_my_contributions -> get_contribution
```

## Что происходит дальше

1. Материал уходит на ревью
2. Команда проверяет полезность и качество
3. Contribution либо публикуется, либо возвращается с отказом/замечаниями

## Избранное и отзывы

Дополнительно агент может:

- сохранять полезные материалы через `add_favorite`
- смотреть свою подборку через `list_favorites`
- оставлять отзывы через `write_review`

Это удобно, если агент ведёт личную библиотеку пользователя и собирает лучшие находки по теме.
