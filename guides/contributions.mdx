# Как делать contributions

Contributions -- это способ поделиться своими наработками с сообществом EdgeLab. Ты можешь отправлять скиллы для агентов, use cases, предложения по событиям. Лучшие работы публикуются на платформе с указанием автора.

## Что можно отправить

| Тип | Описание | Пример |
|---|---|---|
| `skill` | Готовый скилл для AI-агента | Скилл для работы с погодным API |
| `usecase` | Описание практического применения | Как агент автоматизирует утреннюю рассылку |
| `event_suggestion` | Предложение темы для воркшопа или лайва | «Мастер-класс по Scout Pattern» |

## Жизненный цикл contribution

<Steps>

<Step title="Отправка">

Ты создаёшь contribution через API (endpoint `POST /contributions`, coming soon) или через веб-интерфейс платформы.

При отправке contribution получает статус **pending** и ты получаешь +1 поинт.

<Note>
Одновременно может быть не более **2 активных pending** contributions. Дождись ревью предыдущих, прежде чем отправлять новые.
</Note>

</Step>

<Step title="Endorsement (голосование)">

Пока contribution ждёт ревью, другие участники могут за него проголосовать (endorse). Каждый голос приносит тебе +2 поинта.

Endorsements помогают администраторам понять, какие contributions наиболее ценны для сообщества.

</Step>

<Step title="Ревью">

Администратор рассматривает contribution и принимает одно из решений:

| Решение | Что происходит |
|---|---|
| **Approve** | Contribution публикуется на платформе с атрибуцией «by @username». Автор получает +10 поинтов |
| **Reject** | Contribution отклонён. Причина указывается в комментарии |
| **Request revision** | Администратор просит доработать. Можно исправить и отправить повторно |

</Step>

<Step title="Публикация">

Одобренный contribution появляется на платформе. Другие участники могут изучать его, использовать в своих агентах и ссылаться на него.

</Step>

<Step title="Featured (лучший вклад)">

Администратор может отметить contribution как **featured** -- это особый статус для лучших работ. Featured contributions получают дополнительное продвижение и приносят автору +25 поинтов.

</Step>

</Steps>

## Поинты за contributions

| Действие | Поинты |
|---|---|
| Отправка contribution | +1 |
| Каждый endorsement от другого участника | +2 |
| Одобрение (approve) | +10 |
| Статус featured | +25 |

Поинты суммируются. Например, contribution, который получил 5 endorsements, был одобрен и стал featured, принесёт: 1 + (5 × 2) + 10 + 25 = **46 поинтов**.

Все поинты отображаются в [лидерборде](/concepts/points).

## Что делает contribution хорошим

<AccordionGroup>

<Accordion title="Скилл (skill)">

Хороший скилл:
- Решает конкретную задачу (не «делает всё и сразу»)
- Содержит чёткое описание: что делает, какие входы/выходы
- Включает рабочий код с комментариями
- Объясняет, как интегрировать в агента
- Указывает зависимости и требования

</Accordion>

<Accordion title="Use case">

Хороший use case:
- Описывает реальную задачу, которую агент решает
- Показывает архитектуру решения
- Включает примеры взаимодействия (запрос → ответ)
- Объясняет, почему именно такой подход
- Указывает ограничения и edge cases

</Accordion>

<Accordion title="Предложение события (event suggestion)">

Хорошее предложение:
- Чёткая тема, понятная из заголовка
- Описание: чему научатся участники
- Предполагаемый формат (лайв, воркшоп, разбор)
- Примерная длительность
- Целевая аудитория (новички / продвинутые / все)

</Accordion>

</AccordionGroup>

## Revenue sharing (в будущем)

В будущем EdgeLab планирует распределять **20% от выручки** между contributors пропорционально их поинтам. Чем больше полезного контента ты создаёшь -- тем больше будет твоя доля.

Детали программы будут объявлены отдельно.
