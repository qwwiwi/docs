# Как стать контрибьютором

Контрибьюторы -- участники, которые делают EdgeLab лучше: публикуют скиллы, use cases, уроки. Лучшие работы попадают в Knowledge и доступны всему сообществу.

<Info>
Контрибьютором может быть участник с тарифом **Pro** или **VIP**. На тарифе Edge эта функция недоступна.
</Info>

## Что можно предложить

| Тип | Описание | Пример |
|---|---|---|
| **Скилл** | Готовый скилл для AI-агента | Скилл для мониторинга цен через CoinGecko API |
| **Use case** | Описание практического применения | Как агент автоматизирует утренний брифинг |
| **Урок** | Образовательный материал | Пошаговый гайд по Agent-to-Agent коммуникации |

## Требования к submission

Перед отправкой убедись, что у тебя есть:

- **Публичный GitHub-репозиторий** с кодом или материалом
- **Описание проблемы** -- какую задачу решает твой contribution
- **Целевая аудитория** -- для кого: новички, продвинутые, все
- **Рабочий код** (для скиллов) с комментариями и README

## Процесс

<Steps>

<Step title="Отправь submission">

Через API или веб-интерфейс платформы:

<CodeGroup>

```bash curl
curl -X POST https://platform.edgelab.su/api/v1/contributions/submit \
  -H "Authorization: Bearer $EDGELAB_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "skill",
    "title": "Мониторинг цен CoinGecko",
    "description": "Скилл для AI-агента: получение цен, графиков, алертов",
    "repo_url": "https://github.com/username/coingecko-skill",
    "audience": "intermediate"
  }'
```

```python Python
import requests
import os

response = requests.post(
    "https://platform.edgelab.su/api/v1/contributions/submit",
    headers={
        "Authorization": f"Bearer {os.environ['EDGELAB_API_KEY']}",
        "Content-Type": "application/json"
    },
    json={
        "type": "skill",
        "title": "Мониторинг цен CoinGecko",
        "description": "Скилл для AI-агента: получение цен, графиков, алертов",
        "repo_url": "https://github.com/username/coingecko-skill",
        "audience": "intermediate"
    }
)

result = response.json()
print(f"ID: {result['data']['id']}")
print(f"Статус: {result['data']['status']}")
```

</CodeGroup>

При отправке contribution получает статус **pending**.

</Step>

<Step title="Ревью администратором">

Администратор рассматривает submission и принимает одно из решений:

| Решение | Что происходит |
|---|---|
| **Approve** | Материал публикуется в Knowledge с атрибуцией автора |
| **Reject** | Отклонён с указанием причины в комментарии |
| **Revision requested** | Нужна доработка -- можно исправить и отправить повторно |

</Step>

<Step title="Публикация">

Одобренный contribution появляется в Knowledge. Другие участники могут его изучать, использовать в своих агентах и ссылаться на него.

</Step>

</Steps>

## Проверка статуса

Следи за своими contributions через API:

<CodeGroup>

```bash curl
curl -X GET https://platform.edgelab.su/api/v1/contributions/my \
  -H "Authorization: Bearer $EDGELAB_API_KEY"
```

```python Python
response = requests.get(
    "https://platform.edgelab.su/api/v1/contributions/my",
    headers={"Authorization": f"Bearer {os.environ['EDGELAB_API_KEY']}"}
)

for contrib in response.json()["data"]:
    print(f"{contrib['title']} -- {contrib['status']}")
```

</CodeGroup>

Ответ:

```json
{
  "ok": true,
  "data": [
    {
      "id": "ctrb_abc123",
      "title": "Мониторинг цен CoinGecko",
      "type": "skill",
      "status": "pending",
      "submitted_at": "2025-03-15T10:00:00Z",
      "review_comment": null
    }
  ]
}
```

## Вознаграждение

EdgeLab направляет **20% от выручки** в Governance Fund. Средства распределяются между контрибьюторами пропорционально вкладу.

Как это работает:
1. Участники оплачивают подписки (Pro, VIP)
2. 20% от выручки поступает в Governance Fund
3. Контрибьюторы получают долю пропорционально количеству и качеству вкладов

Чем больше полезного контента ты создаёшь -- тем больше твоя доля.

## Что делает contribution хорошим

**Скилл:**
- Решает конкретную задачу (не «делает всё и сразу»)
- Чёткое описание: что делает, входы/выходы, зависимости
- Рабочий код с комментариями

**Use case:**
- Описывает реальную задачу, которую агент решает
- Показывает архитектуру: запрос → обработка → ответ
- Указывает ограничения и edge cases

**Урок:**
- Пошаговая структура: от простого к сложному
- Примеры кода, которые можно запустить
- Целевая аудитория понятна из заголовка

<Info>
Одновременно может быть не более 2 активных pending contributions. Дождись ревью предыдущих, прежде чем отправлять новые.
</Info>
