# Аутентификация

EdgeLab использует MCP, а не REST. Подключение идёт к `https://mcp.edgelab.su/mcp`, а доступ управляется API-ключом формата `el_live_...`.

## Как работает доступ

- Публичные tools можно вызывать без ключа
- Личные tools требуют API-ключ
- Ключ передаётся в заголовке `Authorization: Bearer ...` или `x-api-key`
- JWT для MCP не используется

<Warning>
Если в старых заметках у тебя встречается `edgelab_live_...`, это устаревший формат. Текущий префикс только `el_live_`.
</Warning>

## Формат ключа

API-ключ выглядит так:

```text
el_live_1a8c25f8b7165604
```

Правила:

- Префикс: `el_live_`
- Хвост: `16` hex-символов
- Общая длина: `24` символа

## Где получить ключ

Ключ создаётся через Telegram-бота [@InvestorEdgeLabbot](https://t.me/InvestorEdgeLabbot).

Схема простая:

1. Регистрируешься через бота
2. Бот создаёт аккаунт и связывает `telegram_id` с профилем
3. Бот отправляет API-ключ
4. Этот ключ используешь в MCP-клиенте или своём агенте

<Note>
Веб-регистрации на `api.edgelab.su` для MCP нет. Это старая модель, её не используй.
</Note>

## Заголовки авторизации

<CodeGroup>
```http Authorization
Authorization: Bearer el_live_1a8c25f8b7165604
```

```http x-api-key
x-api-key: el_live_1a8c25f8b7165604
```
</CodeGroup>

Оба варианта валидны. Обычно удобнее `Authorization: Bearer ...`.

## Scopes по тарифам

| Тариф | Scopes | Что это значит |
|---|---:|---|
| Free | 1 | Только `knowledge:read` и публичные гайды |
| Edge | 21 | Полный личный контур: профиль, прогресс, points, support, notifications, events, network, contributions, referrals |
| Pro | 22 | Всё из Edge + `events:write` |
| VIP | 20 | Всё из Pro, кроме `referrals:read` и `referrals:write` |

## Rate limits

| Тариф | Лимит |
|---|---:|
| Free | `30/min` |
| Edge | `60/min` |
| Pro | `120/min` |
| VIP | `300/min` |

Если лимит превышен, MCP-сервер вернёт ошибку `rate_limit_exceeded`. Для циклов и cron-процессов добавь backoff и паузы между вызовами.

## Что доступно с ключом

С ключом открываются личные и социальные tools: профиль, прогресс, achievements, points, subscriptions, auth, events, network, contributions, support, notifications и admin-инструменты для нужных ролей.

## Практические правила

- Не вставляй ключ в URL
- Не коммить ключ в git
- Для каждого агента лучше держать отдельный ключ
- Если ключ засветился, ротируй или удаляй его через auth tools
- Если нужен только публичный поиск по гайдам, ключ вообще не обязателен
