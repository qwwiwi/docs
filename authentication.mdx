# Аутентификация

EdgeLab использует MCP, а не REST. Подключение идёт к `https://mcp.edgelab.su/mcp`. Доступ управляется API-ключом (`el_live_...`) или через OAuth 2.1 с авторизацией через Telegram.


## Два способа аутентификации

| Способ | Когда использовать |
|---|---|
| **API-ключ** (`el_live_...`) | Клиенты с поддержкой заголовков (Claude Desktop, Cursor, Windsurf) |
| **OAuth 2.1 через Telegram** | Клиенты без заголовков (Claude Code CLI), стандартный flow |

Оба способа работают одновременно. Ключ — для клиентов с заголовками. OAuth — для всех остальных.

<Tip>
Подробнее про OAuth: [Подключение через OAuth](/guides/oauth-connect)
</Tip>

## Как работает доступ

- Публичные tools можно вызывать без ключа
- Личные tools требуют API-ключ или OAuth-сессию
- Ключ передаётся в заголовке `Authorization: Bearer ...` или `x-api-key`
- OAuth 2.1 с PKCE — авторизация через Telegram-бота, без паролей и email

<Warning>
Если в старых заметках у тебя встречается `edgelab_live_...`, это устаревший формат. Текущий префикс только `el_live_`.
</Warning>

## Формат ключа

API-ключ выглядит так:

```text
el_live_1a8c25f8b7165604
```

Правила:

- Префикс: `el_live_`
- Хвост: `16` hex-символов
- Общая длина: `24` символа

## Где получить ключ

Ключ создаётся через Telegram-бота [@InvestorEdgeLabbot](https://t.me/InvestorEdgeLabbot).

1. Открой бота и нажми **START**
2. Бот создаёт аккаунт и связывает `telegram_id` с профилем
3. Бот отправляет API-ключ
4. Этот ключ используешь в MCP-клиенте или своём агенте

<Note>
Регистрация и авторизация идут через Telegram. Отдельной веб-регистрации для MCP нет — всё через бота.
</Note>

## OAuth через Telegram

При подключении через OAuth (например, Claude Code):

1. Клиент запускает OAuth flow → открывается браузер
2. На странице — кнопка **«Открыть Telegram»**
3. Нажимаешь → попадаешь в бота → подтверждаешь
4. Страница автоматически перенаправляет обратно → клиент получает токен

Никаких паролей, форм или email-подтверждений. 10–15 секунд.

Подробный гайд: [Подключение через OAuth](/guides/oauth-connect)

## Заголовки авторизации

<CodeGroup>
```http Authorization
Authorization: Bearer el_live_1a8c25f8b7165604
```

```http x-api-key
x-api-key: el_live_1a8c25f8b7165604
```
</CodeGroup>

Оба варианта валидны. Обычно удобнее `Authorization: Bearer ...`.

## Scopes по тарифам

| Тариф | Scopes | Что это значит |
|---|---:|---|
| Free | 1 | Только `knowledge:read` и публичные гайды |
| Edge | 21 | Полный личный контур: профиль, прогресс, points, support, notifications, events, network, contributions, referrals |
| Pro | 22 | Всё из Edge + `events:write` |
| VIP | 20 | Всё из Pro, кроме `referrals:read` и `referrals:write` |

## Rate limits

| Тариф | Лимит |
|---|---:|
| Free | `30/min` |
| Edge | `60/min` |
| Pro | `120/min` |
| VIP | `300/min` |

Если лимит превышен, MCP-сервер вернёт ошибку `rate_limit_exceeded`. Для циклов и cron-процессов добавь backoff и паузы между вызовами.

## Что доступно без ключа

Публичный набор tools:

```text
health, search_knowledge, get_knowledge, list_knowledge
list_categories, get_knowledge_children, list_recent_knowledge
get_sections, get_faq, get_billing_plans
register, email_login, forgot_password, reset_password
agent_forgot_password, agent_reset_password, subscribe, resend_verify
```

## Что доступно с ключом

С ключом открываются личные и социальные tools: профиль, прогресс, achievements, points, subscriptions, auth, events, network, contributions, support, notifications и admin-инструменты для нужных ролей.

## Практические правила

- Не вставляй ключ в URL
- Не коммить ключ в git
- Для каждого агента лучше держать отдельный ключ
- Если ключ засветился, ротируй или удаляй его через auth tools
- Если нужен только публичный поиск по гайдам, ключ вообще не обязателен
