# Система поинтов

Поинты -- это мера вклада участника в сообщество EdgeLab. Ты зарабатываешь поинты, создавая полезные contributions: скиллы, use cases, предложения событий. Поинты отображаются в профиле и в публичном лидерборде.

## Как заработать поинты

| Действие | Поинты | Описание |
|---|---|---|
| Отправка contribution | +1 | За каждый отправленный contribution, независимо от результата ревью |
| Endorsement | +2 | За каждый голос от другого участника за твой contribution |
| Одобрение (approve) | +10 | Contribution прошёл ревью и опубликован |
| Featured | +25 | Contribution отмечен как лучший |

## Пример подсчёта

Ты создал скилл для работы с погодным API:

1. Отправил contribution → **+1** поинт
2. 7 участников проголосовали за (endorsed) → **+14** поинтов
3. Администратор одобрил → **+10** поинтов
4. Через неделю скилл стал featured → **+25** поинтов

Итого: **50 поинтов** за один contribution.

## Лидерборд

Лидерборд -- публичный рейтинг участников по количеству поинтов. Показывает top contributors за:

- Последнюю неделю
- Последний месяц
- Всё время

Лидерборд доступен на платформе и через API (endpoint `GET /contributions/leaderboard`, coming soon).

### Что отображается

| Поле | Описание |
|---|---|
| Ранг | Позиция в рейтинге |
| Имя | username и display_name участника |
| Поинты | Общее количество поинтов за выбранный период |
| Contributions | Количество отправленных contributions |
| Одобрено | Количество одобренных contributions |
| Featured | Количество featured contributions |

## Bounty программа (запланировано)

В рамках bounty программы EdgeLab планирует вознаграждать контрибьюторов за качественные contributions. Программа [запланирована на Q2 2026](/concepts/roadmap).

Как это будет работать:

1. Каждый месяц формируется пул вознаграждений для контрибьюторов
2. Пул распределяется пропорционально поинтам каждого участника
3. Выплаты производятся автоматически

<Note>
Bounty программа находится в стадии планирования. Детали и точные условия будут объявлены отдельно. Поинты, заработанные сейчас, будут учтены при запуске программы.
</Note>

## Стратегия набора поинтов

Несколько советов:

- **Качество важнее количества.** Один одобренный и endorsed contribution даёт больше поинтов, чем пять отклонённых.
- **Решай реальные задачи.** Скиллы и use cases, которые решают конкретные проблемы, получают больше endorsements.
- **Используй feedback.** Если администратор запросил ревизию -- доработай и отправь снова. Это путь к approve.
- **Голосуй за других.** Endorsement -- это не только поинты для автора, но и участие в жизни сообщества.
