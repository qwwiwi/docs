# Путь участника

EdgeLab -- не платформа, где ты просто читаешь уроки. Здесь есть движение: от потребителя контента до создателя, который формирует экосистему.

## Этап 1: Consumer

**Кто:** Все, кто пришёл на платформу.

Ты изучаешь материалы, используешь готовые скиллы и pipeline для своих агентов. Подключаешь Agent API, делаешь первые запросы, разбираешься в архитектуре.

**Что доступно:**
- Вся база знаний (Knowledge API)
- Управление профилем
- Уведомления
- Contributions (просмотр)

## Этап 2: Voter

**Кто:** Участники, которые активно используют платформу.

Ты голосуешь за contributions других участников. Твоими голосами сообщество решает, какие скиллы и use cases достойны публикации. Voters получают 5% от подписок платформы пропорционально активности.

**Что доступно:**
- Всё из Consumer
- Голосование за contributions
- Участие в отборе лучших практик

## Этап 3: Contributor

**Кто:** Специалисты, готовые делиться экспертизой.

Ты создаёшь собственные скиллы, pipeline и use cases. После проверки качества они публикуются с указанием тебя как автора. Contributors получают 20% от подписок платформы пропорционально использованию их работ.

**Что доступно:**
- Всё из Voter
- Submissions API для загрузки contributions
- Статистика использования твоих работ
- Revenue share

## Примеры contributions

| Специальность | Что можно создать |
|---|---|
| Маркетолог | Pipeline автоматизации email-рассылок, skill для анализа конкурентов |
| Дизайнер | Skill для UI-ревью, pipeline генерации мудбордов |
| Разработчик | Skill для code review, шаблон CI/CD агента |
| SMM-специалист | Pipeline планирования контента, skill для аналитики вовлечённости |
| Финансовый консультант | Pipeline анализа портфеля, skill для мониторинга рынков |
| Копирайтер | Skill для проверки текстов, pipeline создания контент-плана |
| Продакт-менеджер | Pipeline приоритизации фичей, skill для анализа фидбэка |
| Аналитик | Pipeline визуализации данных, skill для генерации отчётов |

## Как начать создавать

1. Изучи [требования к contributions](/guides/contributions)
2. Посмотри примеры в каталоге use cases
3. Создай свой skill или pipeline
4. Отправь через Contributions API или веб-интерфейс
5. Дождись проверки и голосования
